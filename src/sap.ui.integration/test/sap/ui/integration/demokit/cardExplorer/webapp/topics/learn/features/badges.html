Badges

Badges can be used to display very short and important information that attracts the user's attention.

Note: Not yet implemented for mobile native cards.

Manifest-defined badges and programmatic badges (added by the card host via customData) can coexist on the same card. Manifest badges are displayed first, followed by programmatic badges. For information on how to add badges programmatically as a card host, see the Badge Integration documentation.

When to Use

Use badges when:

  • The card is new in the user's home page.
  • The card has been updated and the user should be aware of the change.
  • Important status information needs to be communicated at a glance.

Do not use badges if:

  • You want to display the message text 'temporarily unavailable'. Use the disabled state of the card instead.
  • The card should not be visible or used by the user. Don't display the card in the home page in this case.
  • You want to display a counter. Use the numeric header of the card instead.

Behavior and Interaction

By default, badges automatically disappear when the user activates (e.g. clicks, taps) the card, or when the focus is on the card (or the card's content) for more than 3 seconds. This behavior can be changed using the visibilityMode property.

Badge Properties

Badges are defined in the card manifest using the sap.card/badges property, which accepts an array of badge objects. Each badge object supports the following properties:

Property Type Default Value Required Description Schema Version Since
text string "" No The text to display in the badge. Supports data binding. 1.89 1.151
icon string "" No Icon URI (e.g., "sap-icon://status-in-process"). Supports data binding. 1.89 1.151
state string "Indication05" No Color indication. Supports values from IndicationColor (Indication01-20) and ValueState (Success, Error, Warning, Information, None). Supports data binding. 1.89 1.151
visible boolean true No Whether the badge is initially visible. Supports data binding. 1.89 1.151
visibilityMode string "Disappear" No Visibility behavior. "Disappear" hides the badge on card interaction, "Persist" keeps it visible. See CardBadgeVisibilityMode. Supports data binding. 1.89 1.151
announcementText string "" No Custom accessibility announcement text for screen readers. Supports data binding. 1.89 1.151

Examples

Single Badge

{
	"sap.card": {
		"type": "List",
		"badges": [
			{
				"text": "New",
				"icon": "sap-icon://status-in-process",
				"state": "Indication03",
				"visibilityMode": "Persist"
			}
		],
		"header": {
			"title": "Products"
		},
		"content": {
			"item": {
				"title": "{Name}"
			}
		}
	}
}

Multiple Badges

You can define multiple badges that will be displayed side by side:

{
	"sap.card": {
		"type": "List",
		"badges": [
			{
				"text": "3 New",
				"state": "Indication04",
				"visibilityMode": "Persist"
			},
			{
				"icon": "sap-icon://pushpin-off",
				"visibilityMode": "Persist",
				"announcementText": "The card is pinned on top."
			}
		],
		"header": {
			"title": "My Tasks"
		},
		"content": {
			"item": {
				"title": "{Name}"
			}
		}
	}
}

Data Binding

Badge properties support data binding, allowing you to bind badge values to the card's data model:

{
	"sap.card": {
		"type": "List",
		"data": {
			"json": {
				"badgeInfo": {
					"count": "5",
					"status": "New",
					"icon": "sap-icon://status-in-process",
					"color": "Indication03"
				}
			}
		},
		"badges": [
			{
				"text": "{badgeInfo/count} {badgeInfo/status}",
				"icon": "{badgeInfo/icon}",
				"state": "{badgeInfo/color}",
				"visibilityMode": "Persist"
			}
		],
		"header": {
			"title": "Orders"
		},
		"content": {
			"item": {
				"title": "{Name}"
			}
		}
	}
}
