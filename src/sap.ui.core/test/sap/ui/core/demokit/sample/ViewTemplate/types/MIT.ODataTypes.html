Manual Integration Test: OData Types Validation

DESCRIPTION

Test the validation behavior of all OData type input fields with invalid and valid values.

Automated Tests: All manual tests described here are automated in Opa.qunit.js.

PREPARATION

To check console errors open the developer tools before starting the application and check with each step, that there are no errors and warnings.

Testing with mock data

Start the OData Types application URL (adjust host and port if necessary):
  • as sample preview in the SDK: http://<host>:<port>/sapui5-sdk-dist/#/sample/sap.ui.core.sample.ViewTemplate.types/preview
  • local: http://<host>:<port>/test-resources/sap/ui/core/demokit/sample/common/index.html?component=ViewTemplate.types

Testing with back end

Configure your IDE to be able to run the OData Types application against an ABAP back end system. Start the OData Types application URL (adjust host and port if necessary):
  • local: http://<host>:<port>/test-resources/sap/ui/core/demokit/sample/common/index.html?component=ViewTemplate.types&realOData=true

REALIZE and CHECK

General Testing Instructions

Note: The application groups fields by OData type. Each section below corresponds to a visual section in the UI.

How to test fields in tables below:

  1. Locate the field in the UI using the Field Label from the table.
  2. For fields with Invalid Value column: Enter the invalid value, press Enter or Tab. Check: Field shows error state (red border).
  3. Enter the Valid Value, press Enter or Tab. Check: Error state is cleared (border is normal), value is accepted.
  4. For fields with Test Action column: Follow the specific action described and verify the expected result.
  5. Note: Error messages are NOT checked (only error state). Exceptions: Fields with specific error message noted in table.

Special Behaviors:

  • parseEmptyValueToZero: true - Empty string converts to 0.
  • parseKeepsEmptyString: true - Empty string is valid and preserved.
  • Picker controls (DatePicker/DateTimePicker/TimePicker) - Type directly or use picker dialog.
  • Read-only fields - Not editable, not tested.

Buttons: V4 = switch to OData V4 model | Messages = show all errors | Reset Model/Reset ContextBinding = restore original values | Save = save changes (with real OData only)

ADDITIONAL TESTS

Reset and Message Handling

  1. Enter invalid values in multiple fields across different sections.
  2. Check: All validation errors are displayed correctly in their respective fields.
  3. Press the Messages button.
  4. Check: The message popover displays all validation errors.
  5. Correct some errors by entering valid values.
  6. Check: Corrected errors are removed from the message popover.
  7. Press Reset Model or Reset ContextBinding button.
  8. Check: All remaining validation errors are cleared and original values are restored.

Real OData Tests (with Back End)

  1. Start the app with URL parameter realOData=true.
  2. Enter valid values in various input fields across different type sections.
  3. Press the Save button.
  4. Check: Success message is displayed.
  5. Reload the application.
  6. Check: The previously saved values appear in the fields.
  7. Press the V4 button.
  8. Clear the field String40 in the Identification section (leave empty).
  9. Check: The input field is empty.
  10. Press Save.
  11. Check: No error occurs (empty string is valid due to parseKeepsEmptyString).
  12. Reload the application.
  13. Check: The field is still empty.
  14. Press Reset.
  15. Check: Default values provided by the service appear.
  16. Reload the application.
  17. Check: The default values still appear.
